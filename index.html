    Functions, Variables

    Creating Code with Python

    VS Code is a text editor. In addition to editing text, you can visually browse files and run text-based commands at a terminal. In the terminal, you can execute code hello.py to start coding.

    In the text editor, you can type print("hello, world"). This is a famous canonical program that nearly all coders write during their learning process. To run this program, move to the terminal window and type python hello.py.

    Recall that computers only understand zeros and ones. When you run python hello.py, Python interprets your text and translates it for the computer. The result is hello, world printed to the screen. Congrats! You just created your first program.

    Functions

    Functions are pre-defined actions or verbs. In hello.py, the print function knows how to display text in the terminal. Functions take arguments—pieces of data to work with. In print("hello, world"), the string "hello, world" is the argument.

    Bugs

    Bugs are mistakes in your code. They are a natural part of programming, so don't get discouraged! If you typed print("hello, world" (forgetting the last parenthesis), the interpreter would show an error message. These messages are clues to help you fix your code.

    Improving Your First Python Program

    We can make our program interactive using the input function, which takes a prompt as an argument and waits for the user to type something.

    input("What's your name? ")
print("hello, world")

    This alone doesn't use the input. For that, we need variables.

    Variables

    A variable is a container for a value. The equals sign = is the assignment operator; it assigns the value on the right to the variable on the left.

    name = input("What's your name? ")
print("hello,", name)

    Here, the text you type is stored in the name variable, which is then passed as a second argument to print. The output will be hello, David if you type "David".

    Comments and Pseudocode

    Comments, starting with #, are notes for humans that the interpreter ignores. They are crucial for explaining your code's intent.

    Pseudocode is a special type of comment where you plan your program's logic in plain English before writing the actual code.

    # Pseudocode Plan:
# 1. Ask the user for their name
# 2. Print a greeting using that name

# Implementation:
name = input("What's your name? ")
print("hello,", name)

    Strings and Parameters

    A string (str) is a sequence of text. The print function has optional parameters that change its behavior. By default, print uses end='\n', which adds a newline character. We can override this:

    print("hello, ", end="")
print("world")
# Output: hello, world

    Formatting Strings (f-strings)

    The most elegant way to combine variables and text is with f-strings. Start the string with an f and place variables inside curly braces {}.

    name = "David"
print(f"hello, {name}")
# Output: hello, David

    More on Strings (Methods)

    Strings come with built-in functions called methods. You can chain them together for powerful, one-line transformations.

    • .strip(): Removes leading/trailing whitespace.
    • .title(): Capitalizes the first letter of each word.
    name = input("What's your name? ").strip().title()
print(f"hello, {name}")

    Integers and Floats

    An integer (int) is a whole number. A float (float) is a number with a decimal. The input() function always returns a string, so you must convert it if you want to do math.

    Type Casting: Explicitly converting a value from one type to another, like int() or float(). Forgetting to cast numeric input is a common bug!

    x = int(input("What's x? "))
y = int(input("What's y? "))
print(x + y)

    Readability and Formatting

    Code must be readable. Use comments and clear variable names. For formatting numbers, you can use f-strings:

    z = 1000000
print(f"{z:,}")      # Output: 1,000,000

pi = 3.14159265
print(f"{pi:.2f}")  # Output: 3.14

    Defining Your Own Functions with `def`

    Use the def keyword to create your own functions. This helps organize code into reusable blocks. A function can take parameters (inputs) and can have default values.

    # Define a function 'hello' with a parameter 'to' that has a default value
def hello(to="world"):
    print(f"hello, {to}")

# It's conventional to have a 'main' function that runs your program
def main():
    name = input("What's your name? ").strip().title()
    hello(name)  # Calls 'hello' with the user's name
    hello()      # Calls 'hello' using the default value "world"

# This line actually runs the main function
main()

    Returning Values

    Functions can perform an action (like printing) or `return` a value to wherever the function was called. This is essential for building complex programs where results from one function are used as inputs for another.

    def main():
    x = int(input("What's x? "))
    # The value returned by square(x) is printed
    print("x squared is", square(x))

def square(n):
    # This function doesn't print; it sends a value back
    return n * n

main()
